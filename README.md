The Camera Chromaticity & Gamut Mapping Tool is a self-contained, single-file HTML application that visualises the colour gamuts of professional digital cameras and compares them against industry-standard target colour spaces. It requires no server, no installation, and no network connection — open the HTML file in any modern browser.
The tool is built on the CIE 1931 xy chromaticity diagram and presents two complementary plot types: a traditional CIE 1931 xy diagram and an xy polar plot centred on the working white point. Camera characteristics are derived from DNG ForwardMatrix 2 values extracted with ExifTool.

The Colourspace Gamut Mapping Tool is a self-contained, single-file HTML application that visualises industry-standard colour spaces on the CIE 1931 xy chromaticity diagram and polar plot, and applies five distinct gamut mapping strategies to show how colours are transformed when converting from one colour space to another.

The Luminance-Aware Gamut Mapping Tool is a companion to the xy chromaticity gamut mapping tool that adds the luminance dimension missing from pure chromaticity analysis. For any source→target colour space conversion across fifteen colour spaces (D50, D65, and D60 native), it derives the full RGB→XYZ matrix from each space's primary chromaticities and white point using the standard Colour Science method, extracting the Y luminance coefficients that quantify each channel's contribution to overall image brightness — for sRGB these are the canonical 0.2126 / 0.7152 / 0.0722. Primaries and gamut boundaries are then converted to CIELAB and rendered in three complementary views: the a*b* plane showing hue and chroma relationships, the L*C* plane showing lightness against saturation (the most diagnostic view for luminance errors), and the L*h° polar plot combining hue angle with lightness as radius. Crucially, the gamut boundaries are rendered as 120-point sampled polygons rather than three-vertex triangles — this is what actually reveals differences between the five mapping strategies, since primary vertices always lie on the source boundary making per-vertex strategy comparisons degenerate. Each boundary point's luminance is computed from the RGB→XYZ matrix via edge parametrisation, giving physically correct Y values at every point around the gamut outline. The analysis panel reports Y-coefficients for source and target, per-primary CIELAB values (L*, a*, b*, C*, h°) for the target reference, source native, and mapped positions, ΔL* with a perceptual impact rating, and a narrative covering luminance weighting context, per-channel lightness shift implications for skin tone / foliage / sky rendering, mapping outcome and residual error, and a grey balance verdict based on total ΔY across all channels. A structured text export mirrors all three analysis sections.

All are offered "as-is", I hope you find them useful!
