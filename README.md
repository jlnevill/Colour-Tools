The Camera Chromaticity & Gamut Mapping Tool is a self-contained, single-file HTML application that visualises the colour gamuts of professional digital cameras and compares them against industry-standard target colour spaces. It requires no server, no installation, and no network connection — open the HTML file in any modern browser.
The tool is built on the CIE 1931 xy chromaticity diagram and presents two complementary plot types: a traditional CIE 1931 xy diagram and an xy polar plot centred on the working white point. Camera characteristics are derived from DNG ForwardMatrix 2 values extracted with ExifTool.

The Colourspace Gamut Mapping Tool is a self-contained, single-file HTML application that visualises industry-standard colour spaces on the CIE 1931 xy chromaticity diagram and polar plot, and applies five distinct gamut mapping strategies to show how colours are transformed when converting from one colour space to another.

The Luminance-Aware Gamut Mapping Tool is a companion to the xy chromaticity gamut mapping tool that adds the luminance dimension missing from pure chromaticity analysis. For any source→target conversion across sixteen colour spaces spanning D50, D65, and D60 native white points, it derives the full RGB→XYZ matrix from each space's primary chromaticities and white point, extracting the Y luminance coefficients that quantify each channel's contribution to overall brightness — for sRGB these are the canonical 0.2126 / 0.7152 / 0.0722. Primaries and gamut boundaries are converted to CIELAB and rendered in three views selectable via tooltipped buttons: the a*b* plane (hue and chroma), the L*C* plane (lightness vs saturation, C* axis extended to 360 to accommodate ProPhoto's extreme primaries), and L*h° polar (hue angle with L* as radius). Boundaries are sampled at 180 angular intervals from the white point, with each point's luminance derived from the RGB→XYZ matrix via edge parametrisation rather than approximation. Each of the five mapping strategies produces a geometrically distinct Lab representation: Compress shows the uniformly scaled source boundary; Relative Colorimetric shows the WP-translated and clipped boundary; Hard Clip and Hue-Preserve both render the source∩target intersection gamut (they are identical at the boundary — differences only emerge for interior points); and Chroma Map renders a 50% chroma contour illustrating the proportional interior scaling that distinguishes it from the clipping strategies. The analysis panel reports Y-coefficients for both spaces, per-primary CIELAB values (L*, a*, b*, C*, h°) for the target reference, source native, and mapped positions with ΔL* impact ratings, and a narrative covering luminance weighting context, per-channel lightness implications for skin tone, foliage, and sky rendering, strategy-specific mapping outcomes with residual error, and a grey balance verdict based on total ΔY across all channels — all exportable as a structured plain-text report.

All are offered "as-is", I hope you find them useful!
